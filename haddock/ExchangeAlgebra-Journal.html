class (HatVal n, HatBaseClass b, Monoid (a n b)) => Redundant a n b where Source #

Type class for Redundant Algebra. Provides fundamental exchange algebra operations: hat, bar, norm, scalar product, and compress.

Redundant ⊃ Exchange

hat calculation >>> (.^) (10:Not:<Cash .+ 10:Hat:<Deposits) 10.00:Hat:<Cash .+ 10.00:Not:<Deposits

bar calculation >>> x = 10:Not:<Cash .+ 10:Hat:<Deposits >>> y = 5:Hat:<Cash .+ 5:Not:<Deposits >>> (.-) $ x .+ y 5.00:Not:<Cash .+ 5.00:Hat:<Deposits

norm calculation >>> norm $ 10:Not:<Cash .+ 10:Hat:<Deposits 20.0

(.*) calculation >>> (.*) 5 $ 10:Not:<Cash .+ 10:Hat:<Deposits 50.00:Not:<Cash .+ 50.00:Hat:<Deposits

compress calculation >>> compress $ 10:Not:<Cash .+ 5:Hat:<Cash .+ 3:Not:<Cash 5.00:Hat:<Cash .+ 13.00:@Not:<Cash

Minimal complete definition

(.^), (.-), compress, (.+), (.*), norm

Hat operation. Flips Hat/Not on all elements. Complexity: O(1) for singleton, O(n) for Liner (n is the number of base keys)

Bar operation. Cancels Hat/Not on the same base and retains only the difference. Complexity: O(n) (n is the number of base keys)

Alias for bar operation. Identical to (.-).

Aggregates values on the same base. Sums while preserving the Hat/Not distinction. Complexity: O(n) (n is the number of base keys)

Addition of algebra elements. Alias for the Monoid <> operation. Complexity: O(union cost)

Scalar product. Multiplies all element values by a scalar. Complexity: O(1) for singleton, O(n) for Liner

Norm. Returns the sum of all element values. Complexity: O(n) (n is the number of base keys)

data Journal n v b where Source #

Transaction data with annotations. Stored in a base + delta two-layer structure with per-axis indices. Base index is lazy (built on first axis query), while delta index is updated incrementally. Updates are appended only to delta and periodically compacted into base.

insert :: (HatVal v, HatBaseClass b, Note n) => Journal n v b -> Journal n v b -> Journal n v b Source #

Insert x into y. If x's Note already exists in y, it is overwritten with x's value.

>>> type Test = Journal String Double (HatBase AccountTitles)
>>> x = 10.00:@Not:<Cash .| "A" :: Test
>>> y = 20.00:@Not:<Cash .| "B" :: Test
>>> z = 30.00:@Hat:<Cash .| "A" :: Test
>>> insert z (x .+ y)
30.00:@Hat:<Cash.|"A" .+ 20.00:@Not:<Cash.|"B"

projWithNote :: (HatVal v, HatBaseClass b, Note n) => [n] -> Journal n v b -> Journal n v b Source #

projWithNote Projecting with Note.

>>> type Test = Journal String Double (HatBase CountUnit)
>>> x = 1.00:@Hat:<Yen .+ 1.00:@Not:<Amount .| "cat"  :: Test

filterByAxis :: (HatVal v, HatBaseClass b, Note n) => Int -> NoteAxisKey -> Journal n v b -> Journal n v b Source #

Efficiently filter a Journal to entries whose Note matches on the specified axis. Uses base/delta NoteAxisPosting indices for O(|result|) retrieval after index construction.

Axis numbers are 0-indexed. For a Note type (EventName, Term):

  • axis 0 corresponds to EventName
  • axis 1 corresponds to Term

For non-tuple Note types, axis 0 is the only valid axis.

Complexity: O(|result|) after index construction; O(n) for first base-axis query on a Journal value
